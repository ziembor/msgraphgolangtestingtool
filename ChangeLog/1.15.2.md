## [1.15.2] - 2026-01-04

### Changed
- **Expanded Config struct to centralize all configuration** (IMPROVEMENTS.md Code Quality #4.2)
  - Merged separate Flags and Config structs into single comprehensive Config struct
  - Config now contains all 18 configuration fields organized by category:
    - Core configuration (ShowVersion, TenantID, ClientID, Mailbox, Action)
    - Authentication (Secret, PfxPath, PfxPass, Thumbprint)
    - Email recipients (To, Cc, Bcc, AttachmentFiles as stringSlice)
    - Email content (Subject, Body, BodyHTML)
    - Calendar invite (InviteSubject, StartTime, EndTime)
    - Network (ProxyURL)
    - Runtime (VerboseMode, Count)
  - Simplified function signatures - all functions now use single `*Config` parameter instead of `*Flags, *Config`
  - Improved code organization and testability
  - Added inline field comments for all struct fields

- **Updated all function signatures to use unified Config struct**
  - parseAndConfigureFlags() now returns `*Config` instead of `*Flags, *Config`
  - validateConfiguration(config *Config) - single parameter
  - initializeServices(config *Config) - single parameter
  - setupGraphClient(ctx context.Context, config *Config) - removed duplicate parameter
  - executeAction(ctx context.Context, client, config *Config, logger) - single config parameter
  - run() function simplified with unified config handling
