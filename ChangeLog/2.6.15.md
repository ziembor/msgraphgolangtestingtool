# Version 2.6.15

## Changes

### Standardized CSV Logger Error Handling

Fixed inconsistent error handling pattern for csvLogger across the codebase. All 10 files that were using silent error ignoring (`_ = csvLogger.Write...()`) have been updated to use proper error handling with logging.

**Changed from:**
```go
_ = csvLogger.WriteHeader(columns)
_ = csvLogger.WriteRow([]string{...})
```

**Changed to:**
```go
if err := csvLogger.WriteHeader(columns); err != nil {
    logger.LogError(slogLogger, "Failed to write CSV header", "error", err)
}
if logErr := csvLogger.WriteRow([]string{...}); logErr != nil {
    logger.LogError(slogLogger, "Failed to write CSV row", "error", logErr)
}
```
