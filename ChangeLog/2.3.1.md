## Maintenance & Cleanup

### 1. Remove Legacy Version File

**Problem**: The project had two version files - a legacy `src/version.go` that was part of the deprecated build system, and the active `internal/common/version/version.go` used by the current modular structure. The legacy file was:
- Not used by the active build system (`build-all.ps1`)
- Part of the deprecated `go build -C src` build method
- Referenced a non-existent VERSION file via `go:embed`
- Not imported by any active tools (msgraphtool, smtptool)

**Solution**: Removed the legacy `src/version.go` file to eliminate confusion and maintain a single source of truth for version management.

**Changes**:
- Deleted `src/version.go` (legacy file with go:embed VERSION)
- Confirmed all active tools use `internal/common/version/version.go`
- Verified version is correctly displayed by both tools

**Verification**:
```bash
# Both tools show correct version from internal/common/version
msgraphgolangtestingtool.exe -version  # Shows: 2.3.1
smtptool.exe -version                  # Shows: 2.3.1
```

**Benefits**:
- **Single Source of Truth**: Only `internal/common/version/version.go` exists
- **Clear Architecture**: Reflects the project's modular structure (cmd/ + internal/)
- **No Confusion**: Developers know exactly where to update version
- **Simplified Maintenance**: One version file to maintain, not two

### 2. Update Documentation

**Problem**: Documentation still referenced the old version management approach with `src/VERSION` file.

**Solution**: Updated BUILD.md to reflect the current version management approach.

**Changes**:

- Updated version verification instructions

**Files Modified**:

## Technical Details

### Version Management Architecture

The project uses a centralized version management approach:

```go
// File: internal/common/version/version.go
package version

const Version = "2.3.1"

func Get() string {
    return Version
}
```

**Used By**:
- `cmd/msgraphtool/main.go` - Microsoft Graph tool
- `cmd/msgraphtool/config.go` - Configuration handling
- `cmd/smtptool/main.go` - SMTP connectivity tool

**Build Integration**:
- `build-all.ps1` reads version from this file (line 27)
- No go:embed required (version is a compile-time constant)
- Consistent across all tools in the suite

### Migration from Legacy Approach

**Old Approach** (Deprecated):
```go
// src/version.go
package main

//go:embed VERSION
var versionRaw string
var version = strings.TrimSpace(versionRaw)
```

**New Approach** (Current):
```go
// internal/common/version/version.go
package version

const Version = "2.3.1"
```

**Advantages of New Approach**:
1. **Reusable Package**: Can be imported by multiple tools
2. **No External Files**: No VERSION file dependency
3. **Type Safe**: Public constant with clear API
4. **Well Documented**: Clear update instructions in comments
5. **Build Script Integration**: PowerShell can extract version from Go source
