## [1.14.10] - 2026-01-04

### Added
- **Custom flag.Value type for comma-separated lists** (IMPROVEMENTS.md Low Priority #3.2)
  - Created `stringSlice` type implementing `flag.Value` interface
  - Replaced manual `parseList()` calls with idiomatic Go flag parsing
  - Flags `-to`, `-cc`, `-bcc`, and `-attachments` now use custom type
  - Automatic parsing and trimming handled by flag system
  - Added `String()` method for flag value display
  - Added `Set()` method for parsing comma-separated values
  - Added `applyEnvVarsToSlice()` helper for environment variable support

### Changed
- **Improved flag parsing elegance** (IMPROVEMENTS.md Low Priority #3.2)
  - Removed intermediate `toRaw`, `ccRaw`, `bccRaw` string variables
  - Direct use of `stringSlice` types eliminates manual parsing step
  - Flags now declared with `flag.Var()` instead of `flag.String()`
  - Environment variable handling updated for stringSlice flags
  - Cleaner code with fewer intermediate steps

- **Enhanced token security in verbose output** (IMPROVEMENTS.md Low Priority #3.3)
  - Always truncate tokens for security, even if length < 40 characters
  - Short tokens now show maximum 10 characters followed by "..."
  - Prevents accidental exposure of short test tokens
  - Token length always displayed for verification
  - Location: src/msgraphgolangtestingtool.go:994-1006

### Removed
- **Deprecated `parseList()` function** (IMPROVEMENTS.md Low Priority #3.2)
  - No longer needed with custom `stringSlice` type
  - Functionality integrated into `stringSlice.Set()` method
  - Cleaner codebase with less duplication

### Technical Details
- `stringSlice` implements `flag.Value` interface with `String()` and `Set()` methods
- Environment variable mapping updated to exclude stringSlice flags from main `applyEnvVars()`
- Separate `applyEnvVarsToSlice()` function handles stringSlice environment variables
- Token truncation now uses safe bounds checking with `min()` logic
- All recipient and attachment lists processed through idiomatic flag system
