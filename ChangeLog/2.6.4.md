# Version 2.6.4

## Security Enhancements

- Added prominent warning banner when TLS certificate verification is disabled (`-skipverify` flag) in smtptool, pop3tool, and imaptool
- Added `MaskErrorMessage` function to security package for masking credentials in error messages

## Improvements

- Improved context propagation in SMTPClient to support proper cancellation (stored context in struct for use in EHLO, StartTLS, Auth, SendMail methods)

## Bug Fixes

### Code Quality Improvements
- Fixed all golangci-lint errors across the codebase
- Added explicit error handling for intentionally ignored errors (errcheck)
- Removed unused functions: `maskPassword`, `maskAccessToken`, `getTLSVersionString`, `getAttachmentContentBase64`, `isRetryableGraphError`
- Simplified unnecessary nil checks (gosimple)
- Replaced deprecated `tls.VersionSSL30` with raw constant value
- Removed empty if branch in msgraphtool main.go
